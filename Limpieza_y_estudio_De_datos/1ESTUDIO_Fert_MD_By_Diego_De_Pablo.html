1 Introduccin

La estimulacin ovrica es un proceso crucial en tratamientos de fertilidad, y la minera de datos puede desempear un papel fundamental en su optimizacin. En este contexto, exploraremos cmo los patrones ocultos en grandes conjuntos de datos pueden proporcionarnos informacin valiosa sobre incrementar las tasas de xito, las metodologas ms efectivas y otras caractersticas relevantes.

1.1 Caso de estudio

Pacientes que realizan un protocolo de doble estimulacin ovrica con el objetivo de incrementar el nmero de vulos, 60 pacientes siguen la metodologa tradicional mediante el uso de antagonistas de GnRH que inhiben la descarga de LH, a su vez 19 pacientes hacen uso de gestgenos ms econmica y no hace falta inyeccin. La intencin es ver si con uso de gestgenos los resultados son equiparables al protocolo habitual con antagonistas.

1.2 Variables del conjunto de datos:

Si quieres ms informacin puedes acceder a los links, estos llevarn a ms informacin: > Comunes:

  • EDAD: Edad de la paciente.
  • AFC: Recuento de folculos antrales
  • AMH: La Hormona Antimlleriana
  • FACTOR MAS: Presencia/ausencia factor masculino
  • FACTOR FEM: Causa esterilidad femenina
  • BMI: ndice de masa corporal

Las variables usadas tanto en la primera como en la segunda estimulacin:

  • BT AA BB: Nmero de blastos de buena calidad

2.3 Imputacin de datos perdidos

Anteriormente terminamos de manejar los datos perdidos o errneos, ya habindolos catalogados es necesario estimar los valores ausentes en base a los valores vlidos de otras variables y/o casos de la muestra. Esto al ser un paso muy comn en minera de datos se creara dentro de una funcin con intencin de futuras reutilizaciones. Se lograr estimar valores a travs del uso de moda en el caso de variables categricas mientras que para las variables numricas se usar la media. Tambin considerando que uno de los objetivos es comparar la efectividad de ambos grupos de tratamientos se opt por tomar la media y la moda entre su propio grupo para obtener resultados ms congruentes en futuros estudios al remplazar sus respectivos NA con valores correspondientes a su grupo y no al total de pacientes.

rellenado_NA <- function(dataset) {

  # Convertir los valores nulos "" en NA
  dataset[dataset == ""] <- NA
  
  # Identificar las columnas (2) con valores ausentes (anyNA)
  columnas_ausentes <- colnames(dataset)[apply(dataset, 2, anyNA)]

  # Imputar valores faltantes por la media para las variables numricas y la moda para las variables categricas
  dataset[columnas_ausentes] <- lapply(dataset[columnas_ausentes], function(col) {
    if (is.numeric(col)) {
      media <- mean(col, na.rm = TRUE)  # Calcular la media de la columna
      col[is.na(col)] <- media  # Reemplazar los NA con la media
    } else {
      # Convertir las columnas a factores si no lo son
      if (!is.factor(col)) {
        col <- as.factor(col)
      }
      # Calcular la moda de la columna y reemplazar los NA con la moda
      moda <- names(which.max(table(col)))
      col[is.na(col)] <- moda
    }
    return(col)
  })

  # verificacin
  if (anyNA(dataset)) {
    warning("Hubo un error inesperado, por algn motivo no s relleno correctamente")
  }
  return(dataset)
}

Aqu, describimos la variable utilizando estadsticas de resumen. Estas estadsticas incluyen:

  • Medidas de tendencia central, como la media y la mediana, que indican dnde se encuentra el centro de los datos.
  • Medidas de dispersin, como el rango, el rango intercuartlico, la desviacin estndar y la varianza, que describen cun dispersos estn los valores en el conjunto de datos.

calcula_estadis <- function(df) {
  # Filtrar las columnas numricas
  columnas_numericas <- df[, sapply(df, is.numeric)]

  # Calcular estadsticas para cada columna numrica
  estadisticas <- data.frame(
    Mnimo = apply(columnas_numericas, 2, min),
    Mximo = apply(columnas_numericas, 2, max),
    Media = apply(columnas_numericas, 2, mean),
    Mediana = apply(columnas_numericas, 2, median),
    Cuartil_1 = apply(columnas_numericas, 2, function(x) quantile(x, 0.25)),
    Cuartil_3 = apply(columnas_numericas, 2, function(x) quantile(x, 0.75)),
    Rango_Intercuantil = apply(columnas_numericas, 2, IQR),
    Desviacin_Tpica = apply(columnas_numericas, 2, sd),
    Varianza = apply(columnas_numericas, 2, var)
  )

  return(estadisticas)
}

generador_graficas_automatico <- function(dato1, nom_col) {
  es_cat1 <- is.factor(dato1)
  
  if (es_cat1) {
    # Grfico de barras
    barplot(table(dato1), main = paste("Grfico de barras de", nom_col), xlab = nom_col, ylab = "Frecuencia")
    
    # Pastel
    pie(table(dato1), cex = 1.5, main = paste("Pastel de:", nom_col))
  } else {
    # Boxplot
    boxplot(dato1, horizontal = TRUE, main = paste("Boxplot de:", nom_col))
    
    # Histograma
    hist(dato1, main = paste("Histograma de", nom_col), ylab = "Frecuencia", xlab = nom_col)
    
    # Curva de densidad
    densidad <- density(dato1)
    lines(densidad, col = "black", lwd = 2)
    
    # Tabla de frecuencias

4 Anlisis Bivariante

El anlisis bivariante es un mtodo estadstico que examina cmo se relacionan dos variables diferentes. Su objetivo es determinar si existe un vnculo estadstico entre dos variables y, en caso afirmativo, con qu intensidad y en qu direccin.

Al realizar un anlisis bivariante hay un abundante catlogo de herramientas que se pueden aplicar, y al ser un mtodo estadstico fundamental, nos interesar crear una funcin que a partir de 2 variables de un conjunto de dato se determine el tipo de prueba que debe aplicarse, siendo:

  • Chi-cuadrado (chisq.test): Aplicable entre variables categricas, la hiptesis nula: es No hay relacin entre las variables categricas.
  • Fisher (fisher.test): Aplicable entre variables categricas, la hiptesis nula: es No hay diferencia significativa entre las varianzas de las dos muestras.
    • Si el nmero de observaciones en alguna de las celdas de la tabla de contingencia es menor que 5, la aproximacin asinttica del test Chi-cuadrado no se considera vlida, caso en el que la literatura especializada recomienda el uso del test de Fisher.
  • t-test (t.test): Aplicable entre variables categricas o nmericas, la hiptesis nula: es La media de dos poblaciones es igual.
    • La aplicacin de los test estadsticos anteriores tienen limitaciones en cuanto a las distribuciones de los datos. Entre otras, asumen que los datos siguen distribuciones normales, por lo que antes de su aplicacin habr que comprobar esta condicin, para lo que se suele usar el test de Shapiro-Wilk (funcin shapiro.test).
  • Sapphire-test (sapphire.test): la hiptesis nula: es No hay diferencia entre una distribucin normal y la variable.
  • Wilcox-test (Wilcox.test): Aplicable entre variables categricas o nmericas, ideal para comparar las medianas de dos grupos cuando los datos no siguen una distribucin normal. La hiptesis nula: es No hay diferencia entre dos muestras relacionadas.
  • Varianza (var.test): Compara las medias de tres o ms grupos independientes. La hiptesis nula: es La varianza de varias poblaciones son iguales.
  • Kruskal Test (kruskal.test): Compara ms de dos grupos independientes cuando los datos no son normales. La hiptesis nula: es La mediana de varias poblaciones son iguales.
  • Correlacin de Pearson (cor.test(datos,method = "pearson")): Evala la relacin lineal entre dos variables continuas. La hiptesis nula: es No hay correlacin lineal entre dos variables.
  • Correlacin de Spearman (cor.test(datos,method = "spearman")): Evala la relacin monotnica (no necesariamente lineal) entre dos variables. La hiptesis nula: es No hay correlacin montona entre dos variables.

La funcin debera devolvernos el p-valor que es la probabilidad de obtener un estadstico de prueba (como una correlacin o diferencia de medias) suponiendo que la hiptesis nula es cierta. Si el p-valor es muy pequeo (generalmente menor que 0.05 o 0.01), se rechaza la hiptesis nula, lo que sugiere que hay una relacin significativa entre las variables analizadas

#Crear la funcion generica que mide la dependencia entre las variables mediante el p-valor
realizar_analisis <- function(datos1, datos2) {
  # Verificar si los datos son categricos o numricos
  es_cat1 <- is.factor(datos1)
  es_cat2 <- is.factor(datos2)
  
  # Si ambos son categricos
  if (es_cat1 && es_cat2) {
    tabla_contingencia <- table(datos1, datos2)
    if (sum(tabla_contingencia < 5) > 0) {
      fisher_test <- fisher.test(tabla_contingencia)
      return(fisher_test)

    } else {
      chi_test <- chisq.test(tabla_contingencia)
      return(chi_test)
    }
  }
  
  # Si uno es categrico y el otro numrico
  if ((es_cat1 && !es_cat2) || (!es_cat1 && es_cat2)) {
    if (es_cat1) {
      categ_var <- datos1
      num_var <- datos2
    } else {
      categ_var <- datos2
      num_var <- datos1
    }
    
    # Comprobar tamao de la muestra para el test de normalidad
    if (length(num_var) >= 3 && length(num_var) <= 5000) {
      # Comprobar normalidad de la variable numrica
      shapiro_result <- suppressWarnings(shapiro.test(as.numeric(num_var)))
      normalidad <- shapiro_result$p.value > 0.05
      
      # Si la distribucin es normal
      if (normalidad) {
        if (es_cat1) {
          t_test <- t.test(num_var ~ categ_var)
          return(t_test)
        } else {
          anova_test <- aov(num_var ~ categ_var)
          return(anova_test) #CUIDADO!! ESTE ACTUA DIFERENTE A LOS DEMAS
        }
      } else {  # Si la distribucin no es normal
        if (es_cat1) {
          wilcox_test <- wilcox.test(num_var ~ categ_var)
          return(wilcox_test)
        } else {
          kruskal_test <- kruskal.test(num_var ~ categ_var)
          return(kruskal_test)
        }
      }
    } else {
      warning("Hubo un error inesperado, El tamao de la muestra para el test de normalidad debe estar entre 3 y 5000.")
    }
  }
  
  # Si ambos son numricos
  if (!es_cat1 && !es_cat2) {
    # Comprobar tamao de las muestras para el test de normalidad
    if (length(datos1) >= 3 && length(datos1) <= 5000 && length(datos2) >= 3 && length(datos2) <= 5000) {
      # Comprobar normalidad de ambas variables numricas
      shapiro_result1 <- suppressWarnings(shapiro.test(as.numeric(datos1)))
      shapiro_result2 <- suppressWarnings(shapiro.test(as.numeric(datos2)))
      normalidad1 <- shapiro_result1$p.value > 0.05
      normalidad2 <- shapiro_result2$p.value > 0.05
      
      # Si ambas distribuciones son normales
      if (normalidad1 && normalidad2) {
        pearson_corr <- cor.test(as.numeric(datos1), as.numeric(datos2), method = "pearson")
        return(pearson_corr)
      } else {  # Si alguna de las distribuciones no es normal
        spearman_corr <- cor.test(as.numeric(datos1), as.numeric(datos2), method = "spearman")
        return(spearman_corr)
      }
    } else {
      warning("Hubo un error inesperado, El tamao de la muestra para el test de normalidad debe estar entre 3 y 5000.")
    }
  }
  warning("Las variables seleccionadas pueden contener algn fallo, revisa cuidadosamente lo que estas haciendo")
}

filtraPorColumna <- function(dataset, columna) {
  if (columna %in% colnames(dataset)) {
    return(dataset[[columna]]) #equivalente a DATASET$COLUMNA

Generar_tabla <- function(datos) {
  # Crear un data frame vaco
  tabla <- data.frame(Variable1 = character(), Variable2 = character(), ValorP = numeric(), Metodo = character(), stringsAsFactors=FALSE)
  # Obtn todos los nombres de las variables del dataset
  nombres_variables <- names(datos)

  # Genera todas las combinaciones nicas de pares de variables
  combinaciones <- combn(nombres_variables, 2, simplify = FALSE)
  
  # Calcula los valores p para cada par de variables
  resultados <- lapply(combinaciones, function(par) {
    var1 <- par[1]
    var2 <- par[2]
     
    datos1 <- filtraPorColumna(datos, var1)
    datos2 <- filtraPorColumna(datos, var2)
    resultado_completo <- realizar_analisis(datos1, datos2)
    
    resultado <- resultado_completo$p.value
    if (is.null(resultado)) {
      resultado <- 1
      # aov da valores demasiado diferentes, prefiero tratarlos aparte pero no quise alterar mucho la funcin
    }
    test_type <- resultado_completo$method
    if (is.null(test_type)) {
      test_type <- as.character(resultado_completo$call[1])
    }

    # Crear una fila con los resultados
    fila_resultado <- data.frame(Variable1 = var1, Variable2 = var2, ValorP = resultado, Metodo = test_type)
    return(fila_resultado)
  })

  # Combinar todas las filas en un nico DataFrame
  tabla <- do.call(rbind, resultados)
  return(tabla)
}
